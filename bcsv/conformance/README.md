# bcsv conformance fixtures

The directories under [`positive/`](positive/) and [`negative/`](negative/) define a language-agnostic conformance suite for `validate_bcsv` implementations. Every conforming implementation MUST emit identical `code` strings (per spec §4.4) on these fixtures.

## Layout

Each fixture lives in its own directory and contains up to three files:

```
<positive|negative>/<NAME>/
  data.csv          # the CSV payload (omitted for FILE_NOT_FOUND)
  metadata.json     # the paired bcsv metadata (sometimes deliberately malformed)
  expected.json     # the ValidationResult shape any conforming implementation must produce
```

For positive fixtures, `<NAME>` is the datatype being exercised. For negative fixtures, `<NAME>` is the `code` from the §4.4 vocabulary that the fixture must trigger.

## `expected.json` format

```jsonc
{
  // Required: whether ValidationResult.valid must be true or false.
  "valid": true,

  // Required: the set of (code, location) pairs that MUST appear in
  // ValidationResult.errors. Order is not significant; the runner compares as sets.
  "errors": [
    {"code": "SCHEMA_VIOLATION", "location": "/table_schema/columns/0"}
  ],

  // Required: the set of (code, location) pairs that MUST appear in
  // ValidationResult.warnings (when validating with the default `on_violation="warn"`).
  "warnings": [],

  // Optional: validator toggles the runner should pass. Defaults to all checks on
  // with on_violation="warn". A fixture may need (e.g.) check_schema=False to exercise
  // a code that's normally masked by the JSON Schema validation stage.
  "validate_with": { "check_schema": false }
}
```

### Location normalization

Some validation codes (`FILE_NOT_FOUND`, `METADATA_INVALID_JSON`, `HASH_MISMATCH`) include the absolute filesystem path in `location`. The conformance runner compares against the bare filename (`data.csv` or `metadata.json`) by stripping the directory prefix, so `expected.json` uses the bare filename.

Other codes use stable locations: column names (`"age"`), JSON pointers (`"/table_schema/columns/0"`, `"/file_hash"`, `"/dialect"`), or `null` for table-level codes (`COLUMN_ORDER_DIFFERS`, `PRIMARY_KEY_VIOLATION`).

### Message strings are not compared

Validation result `message` fields are implementation-dependent (they include counts, paths, and human-readable English). Only `code` and `location` participate in the parity contract.

## Regenerating the fixtures

The fixtures are generated by [`_generate.py`](_generate.py), which computes `file_hash` SHA-256 digests automatically. Don't edit `.csv` / `.json` files by hand — edit the generator and rerun:

```bash
python bcsv/conformance/_generate.py
```

## Vocabulary covered (17 codes)

Every code in the §4.4 vocabulary has at least one negative fixture:

| Code | Severity | Notes |
|---|---|---|
| `FILE_NOT_FOUND` | error | The `data.csv` file is deliberately absent from this fixture's directory. |
| `METADATA_INVALID_JSON` | error | `metadata.json` contains malformed JSON. |
| `SCHEMA_VIOLATION` | error | Metadata violates `schema.json`'s `allOf` rules. |
| `HASH_MISMATCH` | error | `file_hash` is set to all-zeros — won't match actual CSV. |
| `HASH_ABSENT` | warning | Metadata has no `file_hash` key (integrity not verified). |
| `COLUMN_MISSING_IN_DATA` | error | Metadata declares a column not present in CSV header. |
| `COLUMN_MISSING_IN_METADATA` | error | CSV has a column not declared in metadata. |
| `COLUMN_ORDER_DIFFERS` | warning | Same column set, different order. |
| `LEVELS_REQUIRED` | error | `categorical`/`ordered` missing `levels` (validated with `check_schema=False`). |
| `LEVELS_FORBIDDEN` | error | Non-categorical column has `levels` (validated with `check_schema=False`). |
| `LEVEL_NOT_DECLARED` | warning | A row value is not in the declared `levels`. |
| `RANGE_VIOLATION` | warning | Numeric value outside `[minimum, maximum]`. |
| `LENGTH_VIOLATION` | warning | String length outside `[min_length, max_length]`. |
| `REQUIRED_VIOLATION` | warning | NA in a `required: true` column. |
| `COERCION_FAILED` | warning | Value can't be coerced to declared `datatype`. |
| `PRIMARY_KEY_VIOLATION` | warning | Duplicate rows under declared `primary_key`. |
| `DIALECT_UNSUPPORTED` | warning | `dialect` declares a v0-unsupported key. |

## Datatypes covered (9 positive fixtures)

One per bcsv datatype: `string`, `integer`, `number`, `boolean`, `date`, `datetime`, `time`, `categorical`, `ordered`.
